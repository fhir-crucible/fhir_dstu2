       Field Card. Type and Description
        dispense 0..* Element
      Indicates the details of the dispense event such as the days supply and quantity of a supply dispensed.
        dispense.destination 0..1 Location
      Identification of the facility/location where the Supply was shipped to, as part of the dispense event.
        dispense.identifier 0..1 Identifier
      Identifier assigned by the dispensing facility when the item(s) is dispensed.
        dispense.quantity 0..1 Quantity
      The amount of supply that has been dispensed. Includes unit of measure.
        dispense.receiver 0..* Practitioner
      Identifies the person who picked up the Supply.
        dispense.status 0..1 code
      A code specifying the state of the dispense event.

      Binding: SupplyDispenseStatus(Required)
        dispense.suppliedItem 0..1 Medication | Substance | Device
      Identifies the medication or substance or device being dispensed. This is either a link to a resource representing the details of the item or a simple attribute carrying a code that identifies the item from a known list.
        dispense.supplier 0..1 Practitioner
      The individual responsible for dispensing the medication, supplier or device.
        dispense.type 0..1 CodeableConcept
      Indicates the type of dispensing event that is performed. Examples include: Trial Fill, Completion of Trial, Partial Fill, Emergency Fill, Samples, etc.

      Binding: SupplyItemType(Required)
        dispense.whenHandedOver 0..1 dateTime
      The time the dispensed item was sent or handed to the patient (or agent).
        dispense.whenPrepared 0..1 Period
      The time the dispense event occurred.
        identifier 0..1 Identifier
      Unique identifier for this supply request.
        kind 0..1 CodeableConcept
      Category of supply, e.g. central, non-stock, etc. This is used to support work flows associated with the supply process.

      Binding: SupplyKind(Required)
        orderedItem 0..1 Medication | Substance | Device
      The item that is requested to be supplied.
        patient 0..1 Patient
      A link to a resource representing the person whom the ordered item is for.
        status 0..1 code
      Status of the supply request.
