Observation

  • FHIR Implementation: observation-qicore-qicore-observation
  • FHIR Parent resource: Observation
  • Measurements and simple assertions

    Profile of Observation for decision support/quality metrics. Defines the core set of elements and extensions for quality rule and measure authors.

    • Key:  = Must support,  = QICore-defined extension,  = Is-Modifier

      • Must Support Fields

       Field Card. Type and Description
        applies[x] 0..1 dateTime | Period
      The time or time-period the observed value is asserted as being true. For biological subjects - e.g. human patients - this is usually called the "physiologically relevant time". This is usually either the time of the procedure or of specimen collection, but very often the source of the date/time is not known, only the date/time itself.
        bodySite[x] 0..1 CodeableConcept | BodySite
      Indicates the site on the subject's body where the observation was made ( i.e. the target site).

      Binding: QICorePrecoordinatedBodySite(Preferred)
        code 1..1 CodeableConcept
      Describes what was observed. Sometimes this is called the observation "code".

      Binding: QICoreObservationName(Preferred)
        encounter 0..1 Encounter
      The healthcare event ( e.g. a patient and healthcare provider interaction ) during which this observation is made.
        interpretation 0..1 CodeableConcept
      The assessment made based on the result of the observation.

      Binding: ObservationInterpretation(Extensible)
        issued 0..1 instant
      The date and time this observation was made available.
        method 0..1 CodeableConcept
      Indicates the mechanism used to perform the observation.

      Binding: ObservationMethod(Example)
        reliability 0..1 code
      An estimate of the degree to which quality issues have impacted on the value reported.

      Binding: ObservationReliability(Required)
        status 1..1 code
      The status of the result value.

      Binding: ObservationStatus(Required)
        subject 0..1 Patient | Group | Device | Location
      The patient, or group of patients, location, or device whose characteristics (direct or indirect) are described by the observation and into whose record the observation is placed. Comments: Indirect characteristics may be those of a specimen, fetus, other observer (for example a relative or EMT), or any observation made about the subject.
        value[x] 0..1 Quantity | CodeableConcept | string | Range | Ratio | SampledData | Attachment | time | dateTime | Period
      The information determined as a result of making the observation, if the information has a simple value.

      Binding: QICoreObservationCodedResult(Preferred)
       Field Card. Type and Description
        bodyPosition 0..1 CodeableConcept
      The position of the body when the observation was done, e.g., standing, sitting. To be used only when the body position in not precoordinated in the observation code.

      Binding: QICoreObservationBodyPosition(Preferred)
        comments 0..1 string
      May include statements about significant, unexpected or unreliable values, or information about the source of the value where this may be relevant to the interpretation of the result.
        dataAbsentReason 0..1 CodeableConcept
      Provides a reason why the expected value in the element Observation.value[x] is missing.

      Binding: ObservationValueAbsentReason(Extensible)
        delta 0..1 CodeableConcept
      The qualitative change in the value relative to the previous measurement. Usually only recorded if the change is clinically significant.

      Binding: QICoreObservationDelta(Example)
        device 0..1 Device | DeviceMetric
      The device used to generate the observation data.
        identifier 0..* Identifier
      A unique identifier for the simple observation.
        performer 0..* Practitioner | Organization | Patient | RelatedPerson
      Who was responsible for asserting the observed value as "true".
        referenceRange 0..* Element
      Guidance on how to interpret the value by comparison to a normal or recommended range.
        referenceRange.age 0..1 Range
      The age at which this reference range is applicable. This is a neonatal age (e.g. number of weeks at term) if the meaning says so.
        referenceRange.high 0..1 Quantity
      The value of the high bound of the reference range. If this element is omitted, the high bound of the reference range is assumed to be meaningless. (e.g. reference range is > 5) If the low.comparator element is missing , it is assumed to be '<'.
        referenceRange.low 0..1 Quantity
      The value of the low bound of the reference range. If this element is omitted, the low bound of the reference range is assumed to be meaningless. (e.g. reference range is <2.3) If the low.comparator element is missing, it is assumed to be '>'.
        referenceRange.meaning 0..1 CodeableConcept
      Code for the meaning of the reference range.

      Binding: ObservationRangeMeaning(Example)
        referenceRange.text 0..1 string
      Text based reference range in an observation which may be used when a quantitative range is not appropriate for an observation. An example would be a reference value of "Negative" or a list or table of 'normals'.
        related 0..* Element
      Related observations - either components, or previous observations, or statements of derivation.
        related.target 1..1 Observation
      A reference to the observation that is related to this observation.
        related.type 0..1 code
      A code specifying the kind of relationship that exists with the target observation.

      Binding: ObservationRelationshipType(Required)
        specimen 0..1 Specimen
      The specimen that was used when this observation was made.
