Medication

  • FHIR Implementation: medication-qicore-qicore-medication
  • FHIR Parent resource: Medication
  • Definition of a Medication

    Profile of Medication for decision support/quality metrics. Defines the core set of elements and extensions for quality rule and measure authors.

    • Key:  = Must support,  = QICore-defined extension,  = Is-Modifier

      • Must Support Fields
      • Other Fields
      • All Fields
       Field Card. Type and Description
        code 0..1 CodeableConcept
      A code (or set of codes) that identify this medication. Usage note: This could be a standard drug code such as a drug regulator code, RxNorm code, SNOMED CT code, etc. It could also be a local formulary code, optionally with translations to the standard drug codes.

      Binding: QICoreMedicationCodes(Preferred)
       Field Card. Type and Description
        isBrand 0..1 boolean
      Set to true if the item is attributable to a specific manufacturer (even if we don't know who that is).
        kind 0..1 code
      Medications are either a single administrable product or a package that contains one or more products.

      Binding: MedicationKind(Required)
        manufacturer 0..1 Organization
      Describes the details of the manufacturer.
        name 0..1 string
      The common/commercial name of the medication absent information such as strength, form, etc. E.g. Acetaminophen, Tylenol 3, etc. The fully coordinated name is communicated as the display of Medication.code.
        package 0..1 Element
      Information that only applies to packages (not products).
        package.container 0..1 CodeableConcept
      The kind of container that this package comes as.

      Binding: MedicationContainer(Example)
        package.content 0..* Element
      A set of components that go to make up the described item.
        package.content.amount 0..1 Quantity
      The amount of the product that is in the package.
        package.content.item 1..1 Medication
      Identifies one of the items in the package.
        product 0..1 Element
      Information that only applies to products (not packages).
        product.batch 0..* Element
      Information about a group of medication produced or packaged from one production run.
        product.batch.expirationDate 0..1 dateTime
      When this specific batch of product will expire.
        product.batch.lotNumber 0..1 string
      The assigned lot number of a batch of the specified product.
        product.form 0..1 CodeableConcept
      Describes the form of the item. Powder; tables; carton.

      Binding: MedicationForm(Example)
        product.ingredient 0..* Element
      Identifies a particular constituent of interest in the product.
        product.ingredient.amount 0..1 Ratio
      Specifies how many (or how much) of the items there are in this Medication. E.g. 250 mg per tablet.
        product.ingredient.isActiveIngredient 0..1 boolean
      Flag indicating whether the ingredient is active or inactive.
        product.ingredient.item 1..1 Substance | Medication
      The actual ingredient - either a substance (simple ingredient) or another medication.
        product.usualRoute 0..1 CodeableConcept
      The typical route into the body for this medication.

      Binding: RouteOfAdministration(Example)
