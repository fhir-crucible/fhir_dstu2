Outstanding Issues

There are a number of issues in which input based on implementation experience is specifically sought during the trial use phase:

  • Immunization : Should it be a separate resource or a profile on medication administration?
  • Conformance: How system profiles are used
  • Resource Identification: Managing resource identification in distributed systems
  • Search: How should text search be standardized?
  • Search: How much should _include be standardized?
  • Security Labels: How much policy can be standardized, and how does this impact on this specification?
  • Security: What use cases are there for digital signatures?
  • ClinicalImpression: Content and workflow questions
  • Care Team: Care Team, plus use as a template
  • Data Element: question about specificity
  • Risk Assessment :
  • Messaging: Messaging has not yet been well-exercised at connectathon, so feedback based on implementation is welcome
  • BodySite: Does this need to be a resource, or can it just be a reusable type?

More generally, there are still many outstanding questions about the Request/Fulfillment cycle (Order/ OrderResponse and how various statuses and workflows associated with requests, as well as how proposals cascade to plans to orders to downstream/encoded orders, etc. This is a matter of ongoing investigation.

Deferred for a future version

Some known issues are not addressed in this version of FHIR, and are intended to be addressed in subsequent versions:

  • Adverse Event Reporting
  • An alarm resource to represent current issues with the patient (e.g. device created)
  • Concern Tracking
  • Clinical Studies and Protocols
  • Aggregated Data Reporting including Public Health Reporting
  • Payment related resources, and specifically, an Account resource for payment tracking
  • One or more resources for Advance Care Directive / Power of Attorney
  • Use of RDF
  • A full server side query framework
