Condition

  • FHIR Implementation: condition-qicore-qicore-condition
  • FHIR Parent resource: Condition
  • Detailed information about conditions, problems or diagnoses

    Profile of Condition for decision support/quality metrics. Defines the core set of elements and extensions for quality rule and measure authors.

    • Key:  = Must support,  = QICore-defined extension,  = Is-Modifier

      • Must Support Fields
      • Other Fields
      • All Fields
       Field Card. Type and Description
        abatement[x] 0..1 date | Age | boolean | Period | Range | string
      The date or estimated date that the condition resolved or went into remission. This is called "abatement" because of the many overloaded connotations associated with "remission" or "resolution" - Conditions are never really resolved, but they can abate.
        category 0..1 CodeableConcept
      A category assigned to the condition. E.g. complaint | symptom | finding | diagnosis.

      Binding: ConditionCategory(Required)
        clinicalStatus 1..1 code
      The clinical status of the condition.

      Binding: ConditionClinicalStatus(Required)
        code 1..1 CodeableConcept
      Identification of the condition, problem or diagnosis.

      Binding: QICoreProblemCode(Preferred)
        dateAsserted 0..1 date
      Estimated or actual date the condition/problem/diagnosis was first detected/suspected.
        encounter 0..1 Encounter
      Encounter during which the condition was first asserted.
        location 0..* Element
      The anatomical location where this condition manifests itself.
        location.site[x] 0..1 CodeableConcept | BodySite
      Code that identifies the structural location.

      Binding: QICorePrecoordinatedBodySite(Preferred)
        onset[x] 0..1 dateTime | Age | Period | Range | string
      Estimated or actual date or date-time the condition began, in the opinion of the clinician.
        patient 1..1 Patient
      Indicates the patient who the condition record is associated with.
        severity 0..1 CodeableConcept
      A subjective assessment of the severity of the condition as evaluated by the clinician.

      Binding: DAFProblemSeverity(Required)
        stage 0..1 Element
      Clinical stage or grade of a condition. May include formal severity assessments.
        stage.summary 0..1 CodeableConcept
      A simple summary of the stage such as "Stage 3". The determination of the stage is disease-specific.

      Binding: QICoreConditionStage(Preferred)
       Field Card. Type and Description
        asserter 0..1 Patient | Practitioner
      Person who takes responsibility for asserting the existence of the condition as part of the electronic record.
        criticality 0..1 CodeableConcept
      Characterizes potential impact of the condition on life, durable physiological function, or quality of life. May include concepts such as self-resolving, life-threatening, requires hospitalization, or potential loss of function or capacity. Different from severity in that a moderate subarachnoid hemorrhage is likely to be highly critical, whereas a moderate headache is not as important.

      Binding: QICoreConditionCriticality(Example)

      Further conditions, problems, diagnoses, procedures or events or the substance that caused/triggered this Condition.

      Code that identifies the target of this relationship. The code takes the place of a detailed instance target.

      Binding: ConditionCause(Example)
        dueTo.target 0..1 Condition | Procedure | MedicationAdministration | MedicationStatement | Immunization
      Target of the relationship.
        evidence 0..* Element
      Supporting Evidence / manifestations that are the basis on which this condition is suspected or confirmed.
        evidence.code 0..1 CodeableConcept
      A manifestation or symptom that led to the recording of this condition.

      Binding: QICoreConditionEvidence(Preferred)
        evidence.detail 0..* Resource
      Links to other relevant information, including pathology reports.
        identifier 0..* Identifier
      This records identifiers associated with this condition that are defined by business processed and/ or used to refer to it when a direct URL reference to the resource itself is not appropriate (e.g. in CDA documents, or in written / printed documentation).
        notes 0..1 string
      Additional information about the Condition. This is a general notes/comments entry for description of the Condition, its diagnosis and prognosis.
        occurredFollowing 0..* Element

      Code that identifies the target of this relationship. The code takes the place of a detailed instance target.

      Target of the relationship.
