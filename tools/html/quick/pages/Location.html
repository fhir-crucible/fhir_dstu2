Location

  • FHIR Implementation: location-qicore-qicore-location
  • FHIR Parent resource: Location
  • Details and position information for a physical place

    Profile of Location for decision support/quality metrics. Defines the core set of elements and extensions for quality rule and measure authors.

    • Key:  = Must support,  = QICore-defined extension,  = Is-Modifier

       Field Card. Type and Description
        mode 0..1 code
      Indicates whether a resource instance represents a specific location or a class of locations.

      Binding: LocationMode(Required)
        status 0..1 code
      active | suspended | inactive.

      Binding: LocationStatus(Required)
        type 0..1 CodeableConcept
      Indicates the type of function performed at the location.

      Binding: LocationType(Required)
       Field Card. Type and Description
        address 0..1 Address
      Physical location.
        address.city 0..1 string
      The name of the city, town, village or other community or delivery center.
        address.country 0..1 string
      Country - a nation as commonly understood or generally accepted.

      Binding: USCountry(Extensible)
        address.line 0..* string
      This component contains the house number, apartment number, street name, street direction, P.O. Box number, delivery hints, and similar address information.
        address.period 0..1 Period
      Time period when address was/is in use.
        address.postalCode 0..1 string
      A postal code designating a region defined by the postal service.
        address.state 0..1 string
      Sub-unit of a country with limited sovereignty in a federally organized country. A code may be used if codes are in common use (i.e. US 2 letter state codes).

      Binding: QICore-USPSState(Preferred)
        address.text 0..1 string
      A full text representation of the address.
        address.use 0..1 code
      The purpose of this address.

      Binding: AddressUse(Required)
        description 0..1 string
      Description of the Location, which helps in finding or referencing the place.
        identifier 0..* Identifier
      Unique code or number identifying the location to its users.
        managingOrganization 0..1 Organization
      The organization that is responsible for the provisioning and upkeep of the location.
        name 0..1 string
      Name of the location as used by humans. Does not need to be unique.
        partOf 0..1 Location
      Another Location which this Location is physically part of.
        physicalType 0..1 CodeableConcept
      Physical form of the location, e.g. building, room, vehicle, road.

      Binding: PhysicalType(Example)
        position 0..1 Element
      The absolute geographic location of the Location, expressed in with the WGS84 datum (This is the same co-ordinate system used in KML).
        position.altitude 0..1 decimal
      Altitude. The value domain and the interpretation are the same as for the text of the altitude element in KML (see notes below).
        position.latitude 1..1 decimal
      Latitude. The value domain and the interpretation are the same as for the text of the latitude element in KML (see notes below).
        position.longitude 1..1 decimal
      Longitude. The value domain and the interpretation are the same as for the text of the longitude element in KML (see notes below).
        telecom 0..* ContactPoint
      The contact details of communication devices available at the location. This can include phone numbers, fax numbers, mobile numbers, email addresses and web sites.
