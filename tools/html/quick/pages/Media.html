Media

  • FHIR Parent resource: Media
  • A photo, video, or audio recording acquired or used in healthcare. The actual content may be inline or provided by direct reference

    Base StructureDefinition for Media Resource

    This is a non-QICore profiled resource

    • Key:  = Must support,  = QICore-defined extension,  = Is-Modifier

      • Other Fields
       Field Card. Type and Description
        content 1..1 Attachment
      The actual content of the media - inline or by direct reference to the media source file.
        deviceName 0..1 string
      The name of the device / manufacturer of the device that was used to make the recording.
        duration 0..1 unsignedInt
      The duration of the recording in seconds - for audio and video.
        frames 0..1 positiveInt
      The number of frames in a photo. This is used with a multi-page fax, or an imaging acquisition context that takes multiple slices in a single image, or an animated gif. If there is more than one frame, this SHALL have a value in order to alert interface software that a multi-frame capable rendering widget is required.
        height 0..1 positiveInt
      Height of the image in pixels(photo/video).
        identifier 0..* Identifier
      Identifiers associated with the image - these may include identifiers for the image itself, identifiers for the context of its collection (e.g. series ids) and context ids such as accession numbers or other workflow identifiers.
        operator 0..1 Practitioner
      The person who administered the collection of the image.
        subject 0..1 Patient | Practitioner | Group | Device | Specimen
      Who/What this Media is a record of.
        subtype 0..1 CodeableConcept
      Details of the type of the media - usually, how it was acquired (what type of device). If images sourced from a DICOM system, are wrapped in a Media resource, then this is the modality.

      Binding: DigitalMediaSubtype(Example)
        type 1..1 code
      Whether the media is a photo (still image), an audio recording, or a video recording.

      Binding: DigitalMediaType(Required)
        view 0..1 CodeableConcept
      The name of the imaging view e.g Lateral or Antero-posterior (AP).

      Binding: MediaView(Example)
        width 0..1 positiveInt
      Width of the image in pixels (photo/video).
