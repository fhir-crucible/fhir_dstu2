DeviceUseRequest

  • FHIR Implementation: deviceuserequest-qicore-qicore-deviceuserequest
  • FHIR Parent resource: DeviceUseRequest
  • A request for a patient to use or be given a medical device

    Profile of DeviceUseRequest for decision support/quality metrics. Defines the core set of elements and extensions for quality rule and measure authors.

      Details about this request that were not represented at all or sufficiently in one of the attributes provided in a class. These may include for example a comment, an instruction, or a note associated with the statement.

      Binding: DeviceUseRequestPriority(Required)
        prnReason 0..* CodeableConcept

      Binding: QICoreClinicalFinding(Preferred)
        reasonRejected 0..1 CodeableConcept
      The reason associated with the rejection by the receiver of this device use request. Applicable only if the status is rejected.

      Binding: QICoreActReason(Preferred)
        timing[x] 0..1 Timing | Period | dateTime
      The timing schedule for the use of the device The Schedule data type allows many different expressions, for example. "Every 8 hours"; "Three times a day"; "1/2 an hour before breakfast for 10 days from 23-Dec 2011:"; "15 Oct 2013, 17 Oct 2013 and 1 Nov 2013".
