ProcedureRequest

  • FHIR Implementation: procedurerequest-qicore-qicore-procedurerequest
  • FHIR Parent resource: ProcedureRequest
  • A request for a procedure to be performed

    Profile of ProcedureRequest for decision support/quality metrics. Defines the core set of elements and extensions for quality rule and measure authors.

    • Key:  = Must support,  = QICore-defined extension,  = Is-Modifier

       Field Card. Type and Description
        asNeeded[x] 0..1 boolean | CodeableConcept
      If a CodeableConcept is present, it indicates the pre-condition for performing the procedure.
        bodySite 0..* Element
      Indicates the sites on the subject's body where the procedure should be performed ( i.e. the target sites).
        encounter 0..1 Encounter
      The encounter within which the procedure proposal or request was created.
        indication 0..* CodeableConcept
      The reason why the procedure is proposed or ordered. This procedure request may be motivated by a Condition for instance.
        orderedOn 0..1 dateTime
      The time when the request was made.
        status 0..1 code
      The status of the order.

      Binding: ProcedureRequestStatus(Required)
        subject 1..1 Patient
      The patient who will receive the procedure.
        timing[x] 0..1 dateTime | Period | Timing
      The timing schedule for the proposed or ordered procedure. The Schedule data type allows many different expressions, for example. "Every 8 hours"; "Three times a day"; "1/2 an hour before breakfast for 10 days from 23-Dec 2011:"; "15 Oct 2013, 17 Oct 2013 and 1 Nov 2013".
        type 1..1 CodeableConcept
      The specific procedure that is ordered. Use text if the exact nature of the procedure can't be coded.
