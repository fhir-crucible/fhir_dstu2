Specimen

  • FHIR Implementation: specimen-qicore-qicore-specimen
  • FHIR Parent resource: Specimen
  • Sample for analysis

    Profile of Specimen for decision support/quality metrics. Defines the core set of elements and extensions for quality rule and measure authors.

    • Key:  = Must support,  = QICore-defined extension,  = Is-Modifier

      • Must Support Fields

      • All Fields
       Field Card. Type and Description
        collection.bodySite[x] 0..1 CodeableConcept | BodySite
      Anatomical location from which the specimen was collected (if subject is a patient). This is the target site. This element is not used for environmental specimens.

      Binding: QICorePrecoordinatedBodySite(Preferred)
        collection.collected[x] 0..1 dateTime | Period
      Time when specimen was collected from subject - the physiologically relevant time.
        receivedTime 0..1 dateTime
      Time when specimen was received for processing or testing.
        subject 1..1 Patient | Device | Substance | Group
      Where the specimen came from. This may be from the patient(s) or from the environment or a device.
        treatment.procedure 0..1 CodeableConcept
      A coded value specifying the procedure used to process the specimen.

      Binding: SpecimenTreatmentProcedure(Example)
        type 0..1 CodeableConcept
      The kind of material that forms the specimen.

      Binding: SpecimenType(Example)
       Field Card. Type and Description
        accessionIdentifier 0..1 Identifier
      The identifier assigned by the lab when accessioning specimen(s). This is not necessarily the same as the specimen identifier, depending on local lab procedures.
        collection 0..1 Element
      Details concerning the specimen collection.
        collection.collector 0..1 Practitioner
      Person who collected the specimen.
        collection.comment 0..* string
      To communicate any details or issues encountered during the specimen collection procedure.
        collection.method 0..1 CodeableConcept
      A coded value specifying the technique that is used to perform the procedure.

      Binding: SpecimenCollectionMethod(Example)
        collection.quantity 0..1 Quantity
      The quantity of specimen collected; for instance the volume of a blood sample, or the physical measurement of an anatomic pathology sample.
        collection.quantity.code 0..1 code
      A computer processable form of the units in some unit representation system.
        collection.quantity.isDryWeight 0..1 boolean
      If the recorded quantity of the specimen is reported as a weight, if it is a dry weight.
        collection.quantity.system 0..1 uri
      The identification of the system that provides the coded form of the unit.
        collection.quantity.units 0..1 string
      A human-readable form of the units.
        collection.quantity.value 0..1 decimal
      The value of the measured amount. The value includes an implicit precision in the presentation of the value.
        collection.specialHandling 0..* CodeableConcept
      Special handling during the collection, transport, or storage of the specimen.
        container 0..* Element
      The container holding the specimen. The recursive nature of containers; i.e. blood in tube in tray in rack is not addressed here.
        container.additive[x] 0..1 CodeableConcept | Substance
      Introduced substance to preserve, maintain or enhance the specimen. examples: Formalin, Citrate, EDTA.

      Binding: SpecimenContainerAdditive(Example)
        container.capacity 0..1 Quantity
      The capacity (volume or other measure) the container may contain.
        container.description 0..1 string
      Textual description of the container.
        container.identifier 0..* Identifier
      Id for container. There may be multiple; a manufacturer's bar code, lab assigned identifier, etc. The container ID may differ from the specimen id in some circumstances.
        container.sequenceNumber 0..1 integer
      An assigned number on the specimen denoting the order of collection.
        container.specimenQuantity 0..1 Quantity
      The quantity of specimen in the container; may be volume, dimensions, or other appropriate measurements, depending on the specimen type.
        container.type 0..1 CodeableConcept
      The type of container associated with the specimen (e.g. slide, aliquot, etc).

      Binding: SpecimenContainerType(Example)
        identifier 0..* Identifier
      Id for specimen.
        parent 0..* Specimen
      Reference to the parent (source) specimen which is used when the specimen was either derived from or a component of a another specimen.
        treatment 0..* Element
      Details concerning treatment and processing steps for the specimen.
        treatment.additive 0..* Substance
      Material used in the processing step.
        treatment.description 0..1 string
      Textual description of procedure.
        treatment.treatmentTime 0..1 Period | Duration
      Period or duration of treatment.
