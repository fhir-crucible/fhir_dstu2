DiagnosticOrder

  • FHIR Implementation: diagnosticorder-qicore-qicore-diagnosticorder
  • FHIR Parent resource: DiagnosticOrder
  • A request for a diagnostic service

    Profile of DiagnosticOrder for decision support/quality metrics. Defines the core set of elements and extensions for quality rule and measure authors.

    • Key:  = Must support,  = QICore-defined extension,  = Is-Modifier

      • Must Support Fields
      • Other Fields

       Field Card. Type and Description
        encounter 0..1 Encounter
      An encounter that provides additional information about the healthcare context in which this request is made.
        item.bodySite[x] 0..1 CodeableConcept | BodySite
      Anatomical location where the request test should be performed. This is the target site.

      Binding: QICorePrecoordinatedBodySite(Preferred)
        item.code 1..1 CodeableConcept
      A code that identifies a particular diagnostic investigation, or panel of investigations, that have been requested.

      Binding: QICoreObservationName(Preferred)
        status 0..1 code
      The status of the order.

      Binding: DiagnosticOrderStatus(Required)
        subject 1..1 Patient | Group | Location | Device
      Who or what the investigation is to be performed on. This is usually a human patient, but diagnostic tests can also be requested on animals, groups of humans or animals, devices such as dialysis machines, or even locations (typically for environmental scans).
       Field Card. Type and Description
        clinicalNotes 0..1 string
      An explanation or justification for why this diagnostic investigation is being requested.
        event 0..* Element
      A summary of the events of interest that have occurred as the request is processed. E.g. when the order was made, various processing steps (specimens received), when it was completed.
        event.actor 0..1 Practitioner | Device
      The person who was responsible for performing or recording the action.
        event.dateTime 1..1 dateTime
      The date/time at which the event occurred.
        event.description 0..1 CodeableConcept
      Additional information about the event that occurred - e.g. if the status remained unchanged.

      Binding: QICoreDiagnosticOrderEventDescription(Preferred)
        event.status 1..1 code
      The status for the event.

      Binding: DiagnosticOrderStatus(Required)
        identifier 0..* Identifier
      Identifiers assigned to this order by the order or by the receiver.
        item 0..* Element
      The specific diagnostic investigations that are requested as part of this request. Sometimes, there can only be one item per request, but in most contexts, more than one investigation can be requested.
        item.event 0..* Element
      A summary of the events of interest that have occurred as this item of the request is processed.
        item.precondition 0..* CodeableConcept
      The condition or state of the patient, prior or during the diagnostic procedure or test, for example, fasting, at-rest, or post-operative. This captures circumstances that may influence the measured value and have bearing on the interpretation of the result.

      Binding: QICoreDiagnosticOrderPrecondition(Preferred)
        item.specimen 0..* Specimen
      If the item is related to a specific specimen.
        item.status 0..1 code
      The status of this individual item within the order.

      Binding: DiagnosticOrderStatus(Required)
        orderer 0..1 Practitioner
      The practitioner that holds legal responsibility for ordering the investigation.
        priority 0..1 code
      The clinical priority associated with this order.

      Binding: DiagnosticOrderPriority(Required)
        reason 0..* CodeableConcept
      The reason for this order, as a code. This is often for billing purposes. May relate to the resources referred to in supportingInformation, or code the information in the clinicalNotes field.

      Binding: QICoreProblemCode(Preferred)
        reasonRejected 0..1 CodeableConcept
      The coded reason why the diagnostic order was rejected. The element must be used only when status is rejected.

      Binding: QICoreActReason(Preferred)
        specimen 0..* Specimen
      One or more specimens that the diagnostic investigation is about.
        supportingInformation 0..* Observation | Condition | DocumentReference
      Additional clinical information about the patient or specimen that may influence test interpretations.
